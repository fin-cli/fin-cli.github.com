Happy release day!

Today, I'm excited to bring you FIN-CLI v0.25.0. Check out the [newly published roadmap](/docs/roadmap/) for details on upcoming releases and product focus (hint: there's a future where FIN-CLI no longer supports PHP 5.3).

Let's dive in.

### Compatibility with FinPress 4.7

FinPress 4.7 introduces a new `FIN_Hook` implementation for registering and executing actions and filters. Because FIN-CLI has its own `FIN_CLI::add_fin_hook()` that was erroneously accessing the `$fin_filter` global even when the `add_filter()` function was available, FIN-CLI could fatal in certain circumstances. FIN-CLI now appropriately calls `add_filter()` when it's available.

**Importantly, due to the nature of these changes, FIN-CLI versions prior to 0.25.0 will be incompatible with FinPress 4.7.**

Inspect the change in [this pull request](https://github.com/fin-cli/fin-cli/pull/3384).

### More ways to install FIN-CLI packages

Although we'd love to see your package listed in the Package Index, we realize there are reasons you might not be able to do so. `fin package install` now supports installing an arbitrary Git URL [[#3482](https://github.com/fin-cli/fin-cli/pull/3482)], `.zip` file [[#3485](https://github.com/fin-cli/fin-cli/pull/3485)], or directory path [[#3484](https://github.com/fin-cli/fin-cli/pull/3484)] as a package.

    $ fin package install git@github.com:runcommand/doctor.git
    $ fin package install https://github.com/runcommand/doctor.zip
    $ fin package install doctor

(doctor is the [second premium FIN-CLI command from runcommand](https://runcommand.io/fin/doctor/))

It's worth noting Composer's behavior is slightly different for each package type:

* Git URLs are treated as VCS repositories, and cloned to `~/.fin-cli/packages/vendor`.
* ZIP archives (remote and local) are extracted to `~/.fin-cli/packages/local`, and added as path repositories.
* Local directory paths are added as path repositories, which means Composer creates a symlink to the existing directory path. If the directory you've provided is removed, then the installation will break.

### Everything else in 0.25.0

New commands:

* `fin db check` - Runs `mysqlcheck` with the default `--check` option [[#3332](https://github.com/fin-cli/fin-cli/pull/3332)].
* `fin site option *` - CRUD commands for managing FinPress site options [[#3386](https://github.com/fin-cli/fin-cli/pull/3386)].
* `fin user session *` - CRUD commands for managing user sessions [[#3307](https://github.com/fin-cli/fin-cli/pull/3307)].

Command improvements:

* `fin core language update`:
   * Fixes strict standard error about variable reference [[#3380](https://github.com/fin-cli/fin-cli/pull/3380)].

* `fin core multisite-(install|convert)`:
   * Warns when multisite constants can't be inserted into `fin-config.php`, instead of erroneously inserting at the end [[#3272](https://github.com/fin-cli/fin-cli/pull/3272)].

* `fin core update-db`:
   * Ensures `fin core update-db --network --dry-run` is actually dry [[#3347](https://github.com/fin-cli/fin-cli/pull/3347)].

   * Adds a simple progress indicator [[#3407](https://github.com/fin-cli/fin-cli/pull/3407)].

* `fin scaffold plugin-tests`:
   * Uses PHP version specific to Trusty on CircleCI [[#3359](https://github.com/fin-cli/fin-cli/pull/3359)].

   * Prevents error notice when `export_insert_size` isn't defined [[#3357](https://github.com/fin-cli/fin-cli/pull/3357)].
